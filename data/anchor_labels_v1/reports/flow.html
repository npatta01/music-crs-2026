How we turn raw conversations into clean training labels

A pipeline to teach a music recommender to actually follow what the listener just asked for.

The problem we're fixing ("anchoring"). When a listener says "play something by a different artist," today's recommender often replies with… the same artist again. It ignores the pivot. We're building data that teaches it not to.
The catch: the raw data is AI-generated and comes with a "liked / disliked" tag we don't fully trust — so independent AI judges re-check every recommendation.

Step 1 — The input (one turn)

What the system looks at

the last ~3 turns (incl. the current ask) + what was played each turn + the one track being judged

The recent conversation — not just the current line — so we can tell if the listener has been asking for the same thing repeatedly. We do this for all 106,393 turns.

▼

Step 2 — What we judge (two separate questions)

Did the track honor the current request?

① Artist pivot the main bug

"Did the listener ask for a DIFFERENT artist?"
yesno
＋ a 100%-reliable check: is the played track the same artist as before? (exact catalog match — no AI guessing)

Asked for different + got same = ANCHORING.

② Content fit

"Does the track match the named genre / era / mood / song?"
valid invalid unsure

Covers pivots on any facet — e.g. "no more high-energy rock."

▼

Step 3 — Who answers

Two cheap AI judges vote; a strong judge breaks ties

✅ They agree → trust it

Two different AI models answer both questions. When they agree (most of the time), we take the answer.

⚖️ They disagree → ask Opus

On the ~8% where they conflict, the strongest, most trusted judge (Claude Opus) re-judges the case independently — it answers the same two questions fresh, it does not just pick a side.

The two cheap judges (Gemma + DeepSeek-V4-Flash, different model families) get the identical prompt — only the model differs. Why not Opus for everything? Cost: the cheap judges cover all 106k turns; Opus is reserved for the hard, disputed cases.

▼

Step 4 — The label (and how the judgment decides it)

Each turn becomes one labeled example + a confidence weight

NEGATIVE Asked for a different artist but got the same one → the anchoring bug. Marked negative even if the data said "liked."
NEGATIVE Track clearly violates the named request (wrong genre/era, or carries something they just rejected).
POSITIVE Track fits the request and the listener liked it → the good example to learn from.
DROP Track fit the request but was disliked → personal taste, ambiguous → set aside (low weight).
HOLD Can't verify (subtle mood, unrecognized song) → low-confidence, defer to the original tag.
