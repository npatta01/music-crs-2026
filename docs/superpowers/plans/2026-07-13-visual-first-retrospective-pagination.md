# Visual-First Retrospective Pagination Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the Music-CRS retrospective as a visual-first, two-dimensional deck with a 50-page baseline, seven horizontal chapters, responsive evidence layouts, and no nested vertical scrolling.

**Architecture:** Keep the canonical report blocks, payload, URLs, sources, and iframe source documents unchanged in the portable fallback. Extend the deterministic Node enhancer with an exhaustive page manifest, locally embedded visual assets, page archetypes, responsive iframe sizing, and legacy-hash aliases; use browser tests to prove that the enhanced view preserves all evidence and works at the target viewports.

**Tech Stack:** Node.js ESM and `node:test`; browser-native HTML/CSS/JavaScript and Shadow DOM; Python 3.12, pytest, Playwright, and installed Google Chrome; Git.

## Global Constraints

- Use seven horizontal chapters and a 50-page baseline; permit later evidence-driven page splits without changing the navigation model.
- Preserve all 74 canonical blocks exactly once, ten sources, evidence URLs, commit-pinned acknowledgements, calculations, source documents, and compressed payload.
- Do not modify canonical factual conclusions or evidence boundaries in `artifact.json` or `evidence.json`.
- Use one page for one idea and avoid intimidating text walls; conceptual pages lead with imagery or a deterministic visual.
- Use one vertical scroller per chapter. No iframe, disclosure, audit, card, or diagram may add a nested vertical scrollbar.
- Keep the static, JavaScript-disabled, print, and `?view=linear` report complete and in canonical block order.
- Use deterministic, labeled HTML diagrams to explain systems and subsystems; decorative abstract imagery is out of scope.
- Keep all mechanisms, qualifications, scores, repository names, and evidence in accessible native HTML.
- Do not add runtime dependencies, CDN assets, external fonts, analytics, remote calls, or public deployment.
- Preserve readable hashes, back/forward history, keyboard navigation, Jump search, reduced motion, forced colors, 44px touch targets, and existing hash aliases.
- At 1440×900, 1280×800, 1024×768, and 390×844, prevent clipped evidence, unintended document-level horizontal overflow, and internal vertical scrollbars.

## File Structure

`scripts/report/retrospective_deck.mjs`
: Owns the 50-page manifest, page archetypes, asset manifest, validation, legacy-hash aliases, deterministic injection, responsive iframe sizing, navigation, and CLI.

`tests/report/retrospective_deck.test.mjs`
: Tests the manifest, exact block assignment, structured explanatory flows, aliases, deterministic enhancement, and immutable canonical payload.

`tests/report/test_retrospective_deck_browser.py`
: Tests rendered page count, navigation, intrinsic iframe sizing, overflow, responsive matrices, accessibility, fallback modes, and target viewports.

`retrospective.html`
: Remains the only reader-facing artifact and is regenerated by the deterministic enhancer.
